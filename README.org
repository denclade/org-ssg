An Emacs static site generator built for speed and a seamless developer experience. 
Write your content in org-mode, configure everything in a simple local file, and let Emacs handle the rest.

*Features:*
- **Zero-config defaults:** Auto-detects paths based on project root.
- **Hierarchical Virtual Indices:** Deeply nested folders (e.g., =videos/inf/webdev/=) automatically generate smart, pre-filtered index pages—no manual configuration required!
- **Advanced Native Template Engine:** Built-in support for nested ={{#each}}= loops with scoping (=as item=), nested ={{#if}}= conditions, dot-notation (=post.title=), and custom Lisp filters. 100% Emacs-native, zero external dependencies.
- **Live-Reload & Watcher:** Save a file in Emacs, see it instantly in the browser.
- **Native SCSS Support:** Automatically compiles =.scss= files when sass exists on your system.
- **Rich Excerpts:** Native support for =#+begin_excerpt= blocks.
- **Custom Link Types:** Beautiful buttons and icon links out of the box.

/Note: This package is a heavily modified fork of the original =org-grimoire= by Strahinja Piperac./

** Installation

Clone the repository and add it to your load path:

#+begin_src elisp
(add-to-list 'load-path "~/path/to/org-ssg/")
(require 'org-ssg)
#+end_src

** Configuration

org-ssg uses a project-local configuration file. Create a file named =.ssg.el= in the root directory of your site. It contains a simple property list (plist):

#+begin_src elisp
(:site-title   "My Org Site"
 :base-url     "https://example.com"
 :description  "A site built with Emacs"
 :theme        "default"
 :per-page     10
 :type-aliases (("posts" . "post")
                ("videos" . "video"))
 :filters      (("truncate" . (lambda (val)
                                (if (> (length val) 50)
                                    (concat (substring val 0 47) "...")
                                  val)))
                ("shout"    . upcase)))
#+end_src

org-ssg automatically derives all essential paths from the directory where =.ssg.el= is located. 

| Key          | Default Derived Path         | Description                                     |
|--------------+------------------------------+-------------------------------------------------|
| =:source=    | =base-dir/content/=          | Where your .org files live (accepts a list)     |
| =:output=    | =base-dir/public_html/=      | Generated HTML output                           |
| =:static=    | =base-dir/static/=           | Static assets like CSS/JS (accepts a list)      |
| =:theme=     | =base-dir/themes/default/=   | Path to your HTML templates                     |
| =:filters=   | =nil=                        | Alist of custom Lisp functions for templates    |

** Development Workflow (Live-Reload)

org-ssg provides a modern developer experience right out of Emacs.

1. Navigate to your project directory.
2. Run =M-x org-ssg-serve=. This will start a local web server (=http://localhost:8080=) and automatically open your default browser.
3. Run =M-x org-ssg-watch=. This attaches a file watcher to your source, static, and theme directories.

Now, whenever you save a file in Emacs, org-ssg will automatically rebuild the site and **hot-reload your browser**. 
To stop the watcher, run =M-x org-ssg-stop-watch=.

** Writing Content

The content type is automatically inferred from the directory the file lives in. org-ssg supports **multiple types** based on folder depth. 
For example, a file in =content/videos/inf/webdev/post.org= will automatically be assigned the types: =videos=, =inf=, and =webdev=.

#+begin_src org
#+TITLE: My Post
#+DATE: 2026-05-28
#+TAGS: emacs lisp
#+CSS: /static/css/gallery.css
#+MY_CUSTOM_VAR: Hello World

#+begin_excerpt
This is a rich-text excerpt. It will be rendered on the index page!
#+end_excerpt

Post content goes here. Full org-mode syntax supported.
#+end_src

*** Frontmatter keywords

| Keyword         | Description                                  | Example                     |
|-----------------+----------------------------------------------+-----------------------------|
| =#+TITLE:=      | Page title                                   | =#+TITLE: My Post=          |
| =#+DATE:=       | Publication date (yyyy-mm-dd)                | =#+DATE: 2026-05-28=        |
| =#+TYPE:=       | Override the inferred folder type(s)         | =#+TYPE: page, special=     |
| =#+TEMPLATE:=   | Force a specific HTML template               | =#+TEMPLATE: landing-page=  |
| =#+COMPACT:=    | Custom var often used for list layouts       | =#+COMPACT: t=              |
| =#+TAGS:=       | Space/comma separated list of tags           | =#+TAGS: emacs lisp=        |
| =#+EXCERPT:=    | A plain-text summary (fallback for blocks)   | =#+EXCERPT: A short read.=  |
| =#+CSS:=        | Inject specific CSS into the document <head> | =#+CSS: /static/custom.css= |
| =#+JS:=         | Inject specific JS into the document <head>  | =#+JS: /static/script.js=   |
| =#+COLLECTION:= | Generate a manual index with AND-logic       | =#+COLLECTION: videos, inf= |
| =#+DRAFT:=      | Set to =t= to completely exclude from build  | =#+DRAFT: t=                |
| =#+LISTED:=     | Set to =nil= to hide from index listings     | =#+LISTED: nil=             |

*Note on Custom Keywords:* Any unrecognized keyword (like =#+MY_VAR: value=) is automatically extracted and passed to the template engine. You can render it via ={{my-var}}=.

*** Virtual Indices

org-ssg will automatically generate "virtual" index pages for every subfolder in your =content/= directory, so you don't have to write manual =index.org= files.

If you have a file at =content/videos/inf/post.org=:
- org-ssg auto-generates =/videos/index.html= (listing ALL videos).
- org-ssg auto-generates =/videos/inf/index.html= (listing ONLY posts that are both videos AND inf).

Virtual indices are automatically passed the variable =compact= (=t=) and use the =posts.html= template, making it incredibly easy to style them dynamically.

/Want to override a virtual index?/ Just create an =index.org= inside that folder. It will seamlessly replace the virtual one!

*** Custom Links & Buttons
org-ssg provides custom link types to easily add CSS classes to your inline links:

#+begin_src org
[[btn:/download][Download Now]]
[[urltool:/tools][Toolbox]]
[[urlexternal:https://github.com][GitHub]]
#+end_src
These render as:
- =<a href="..." class="button primary">=
- =<a href="..." class="with-icon box">=
- =<a href="..." class="with-icon external">=

*** Static Assets & Automatic SCSS

Any file placed in your =static/= directory is copied to =public_html/static/=. 

*Bonus:* If you have a SCSS compiler installed on your system (=sass=, =sassc=, or =node-sass=), org-ssg will automatically compile any =.scss= files to =.css= on the fly. Emacs temporary lockfiles are safely ignored.

You can link them directly in your org files:
#+begin_src org
#+CSS: /static/css/style.scss
#+end_src
org-ssg will compile it and rewrite the HTML link to point to =style.css=.

** Templates

org-ssg features a powerful, Emacs-native template engine. It supports includes, scoped loops, deeply nested conditionals, and Lisp-powered filters.

*** 1. Template Resolution

Templates are matched based on the content =type= (or the =#+TEMPLATE:= override):
1. Explicit =#+TEMPLATE: custom= -> searches for =custom.html=
2. Alias matching: If type is =videos= and =("videos" . "video")= is in =:type-aliases= -> searches for =video.html=
3. Fallback: Searches for =page.html=

*** 2. Variables, Filters & Dot-Notation

Access standard or custom frontmatter variables using double brackets. You can pipe variables through filters using the =|= symbol.

#+begin_src html
<h1>{{title}}</h1>
<time>{{date | date}}</time>
<p>{{post.excerpt | truncate}}</p>
#+end_src

Built-in filters are =date=, =upcase=, and =downcase=. Add your own in =.ssg.el=.

*** 3. Deeply Nested Conditionals (If/Else)

You can conditionally render blocks based on variable existence or truthiness, and safely nest them:

#+begin_src html
<article>
  {{#if draft}}
    <span class="warning">Draft</span>
  {{else}}
    <span class="success">Published</span>
    {{#if custom_video_url}}
      <iframe src="{{custom_video_url}}"></iframe>
    {{/if}}
  {{/if}}
</article>
#+end_src

*** 4. Scoped Loops (Lists)

Iterating over collections is done with ={{#each}}=. You can use the =as= keyword to scope your variables (e.g., =post.title=), which prevents variable collisions and keeps your templates perfectly clean!

#+begin_src html
<ul class="post-list">
  {{#each posts as post}}
    <li>
      {{#if compact}}
        <div class="metadata-row">
          <time>{{post.date | date}}</time>
          {{#if post.reading-time}}
            <span>{{post.reading-time}} Min.</span>
          {{/if}}
        </div>
        <a href="{{post.url}}">{{post.title}}</a>
      {{else}}
        <article class="post-item">
          <h2><a href="{{post.url}}">{{post.title}}</a></h2>
          <div class="post-abstract">{{post.excerpt}}</div>
        </article>
      {{/if}}
    </li>
  {{/each}}
</ul>

{{pagination}}
#+end_src

*** 5. Base template & Includes

=base.html= is the site-wide layout. All single pages and index pages are wrapped in it. You can inject external files using ={{include file.html}}=.

#+begin_src html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>{{title}}</title>
  {{extra-css}}
</head>
<body>
  {{include partials/navbar.html}}
  
  <main>
    {{content}}
  </main>
  
  {{include partials/footer.html}}
  {{extra-js}}
</body>
</html>
#+end_src

*** Available Placeholders (Standard Variables)

These are the built-in variables available in your templates. Any custom frontmatter keywords (e.g., =#+MY_VAR:=) are also available as placeholders (e.g., ={{my-var}}=).

| Placeholder      | Description                                      |
|------------------|--------------------------------------------------|
| ={{title}}=      | Current page title                               |
| ={{site-title}}= | Global site title from =.ssg.el=                 |
| ={{content}}=    | Rendered page content (HTML)                     |
| ={{excerpt}}=    | Post excerpt (used in lists)                     |
| ={{url}}=        | Root-relative URL of the current page            |
| ={{date}}=       | Publication date (raw format, use filter!)       |
| ={{tags}}=       | Rendered HTML tag links                          |
| ={{extra-css}}=  | CSS links injected from =#+CSS:=                 |
| ={{extra-js}}=   | JS tags injected from =#+JS:=                    |
| ={{compact}}=    | Boolean =t= (auto-passed to virtual indices)     |

** Feeds
RSS and Atom feeds are automatically generated at =/rss.xml= and =/atom.xml= from all published posts. A sitemap is generated at =/sitemap.xml=.

** Whats going on?
1. You start with =org-ssg-build=
2. It searches for all .org files, recursively from root
3. It derives types based on folder depth (e.g. =videos/inf=)
4. It dynamically generates virtual paginated indices for folders missing an =index.org=
5. It renders the site using the native, depth-aware template engine
6. It copies required / referenced static assets & compiles SCSS
